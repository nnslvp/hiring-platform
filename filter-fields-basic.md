FILTER_FIELDS_BASIC
================================================================================

ОПИСАНИЕ:
  Базовый набор полей для фильтрации водителей в базе данных вакансий.
  Эти поля позволят быстро находить подходящих кандидатов по ключевым критериям.

================================================================================

НОВЫЕ ПОЛЯ ДЛЯ ДОБАВЛЕНИЯ В БАЗУ ДАННЫХ:

FIELD_1: Категория прав
  TYPE: multi_select
  DESCRIPTION: Категории водительских прав, требуемые для вакансии
  SOURCE: Блок "Водительские права:" в разделе "Требования к кандидату"
  OPTIONS:
    - B (легковой автомобиль)
    - C (грузовик)
    - C1 (грузовик малой грузоподъемности)
    - CE (грузовик с прицепом)
    - D (автобус)
    - D1 (малый автобус)
  RULES:
    - Извлечь все категории из текста
    - Разделить по запятым, если несколько
    - Убрать слово "Категория" и пробелы
    - "C+E", "C + E", "C/E" → "CE"
    - ВАЖНО: BDF — это ТИП ТЕХНИКИ, а НЕ категория прав! Не путать!
  USAGE: Позволяет фильтровать вакансии по требуемым категориям прав
  EXAMPLES:
    "Категория CE" → [CE]
    "Категория C+E" → [CE]
    "Категория CE, BDF" → [CE] (BDF идёт в "Тип техники"!)
    "Категория B" → [B]

FIELD_2: Тип техники
  TYPE: multi_select
  DESCRIPTION: Тип транспортного средства и прицепа
  SOURCES:
    - Блок "Прицеп:" в разделе условий работы
    - Блок "Автомобиль:" в разделе условий работы
    - Название вакансии (может содержать тип техники в скобках)
  OPTIONS:
    - Тент
    - Реф (рефрижератор)
    - BDF
    - Контейнеры
    - Цистерна
    - Изотерма
    - Штора
    - Самосвал
    - Тандем
    - Подконтейнерный прицеп
  RULES:
    - Извлечь тип из блока "Прицеп:"
    - Если в названии вакансии есть тип в скобках - использовать его
    - "Тентованный" → Тент
  USAGE: Фильтрация по типу техники, на которой предстоит работать
  EXAMPLES:
    "Прицеп: BDF" → [BDF]
    "Прицеп: Тентованный" → [Тент]
    "Прицеп: Реф" → [Реф (рефрижератор)]
    Название: "Szczecin (тент)" → [Тент]

  TYPE: multi_select
  DESCRIPTION: Географические регионы, где предстоит работать
  SOURCE: Блок "Регионы работы:" или "Постоянные маршруты/регионы:"
  OPTIONS (базовый список, только русские названия):
    - Польша
    - По всей Европе
    - Скандинавия
    - Великобритания
    - Германия
    - Франция
    - Италия
    - Испания
    - Бенилюкс
    - Чехия
    - Словакия
    - Австрия
    - Венгрия
    - Литва
    - Латвия
    - Эстония
    - Словения
    - Греция
    - Болгария
    - Дания
    - Швеция
    - Финляндия
    - Норвегия
    - Швейцария
    - Румыния
    - Хорватия
    - Босния
    - Сербия
    - Восточная Европа
    - Западная Европа
    - Нидерланды
    - Бельгия
    - Люксембург
    - Португалия
  RULES:
    - ВСЕГДА использовать РУССКИЕ названия стран
    - Входные данные могут быть на польском - переводить на русский
    - "по всей Европе", "EU", "UE", "Европа", "вся Европа", "по ЕС" → "По всей Европе"
    - Если перечислены страны → извлечь каждую страну на русском языке
  USAGE: Фильтрация вакансий по регионам работы
  NOTE: Notion автоматически создает новые опции при добавлении через API, если указанной страны/региона нет в списке. Можно использовать любые страны и регионы, НО ТОЛЬКО НА РУССКОМ ЯЗЫКЕ.
  EXAMPLES:
    "по всей Европе" → [По всей Европе]
    "EU" → [По всей Европе]
    "Niemcy, Francja" → [Германия, Франция]
    "Германия, Чехия, Австрия, Польша" → [Германия, Чехия, Австрия, Польша]

FIELD_4: Минимальный опыт (месяцы)
  TYPE: number
  DESCRIPTION: Минимальный требуемый опыт работы водителем в месяцах
  SOURCE: Блок "Минимальный опыт работы:" в разделе "Требования к кандидату"
  FORMAT: Целое число (0, 6, 12, 24, 36 и т.д.)
  RULES:
    - "Полгода" → 6
    - "6 месяцев" → 6
    - "1 год" → 12
    - "12 месяцев" → 12
    - "2 года" → 24
    - "Желателен 6 месяцев" → 6 (если указано как желательно, все равно указать)
    - Если не указано → null
  USAGE: Фильтрация по минимальному опыту работы
  EXAMPLES:
    "Полгода" → 6
    "Желателен 6 месяцев" → 6
    "1 год" → 12
    "Не указано" → null
  NOTE: Можно использовать для сортировки и фильтрации "больше или равно"

FIELD_5: Код 95
  TYPE: select
  DESCRIPTION: Требование к наличию свидетельства квалификации (код 95)
  SOURCE: Блок "Свидетельство квалификации (код 95):" в разделе "Требования к кандидату"
  OPTIONS:
    - Обязательно
    - Желательно
  RULES:
    - "Обязательно" → Обязательно
    - "Желательно" → Желательно
    - "Не требуется", "Не требуются", не указано → null (пустое = не требуется)
  USAGE: Быстрая фильтрация по требованию к Код 95. null = не требуется
  EXAMPLES:
    "Обязательно" → Обязательно
    "Желательно" → Желательно
    "Не требуется" → null

FIELD_6: ADR
  TYPE: select
  DESCRIPTION: Требование к наличию разрешения на перевозку опасных грузов
  SOURCE: Блок "ADR (опасные грузы):" или "ADR (перевозка опасных грузов):"
  OPTIONS:
    - Обязательно
    - Желательно
  RULES:
    - "Обязательно" → Обязательно
    - "Желательно" или "Желательно, но не обязательно" → Желательно
    - "Не требуется", не указано → null (пустое = не требуется)
  USAGE: Фильтрация вакансий, требующих перевозку опасных грузов. null = не требуется
  EXAMPLES:
    "Обязательно" → Обязательно
    "Желательно, но не обязательно" → Желательно
    "Не требуется" → null

FIELD_7: Тип оплаты
  TYPE: select
  DESCRIPTION: Тип системы оплаты труда
  SOURCE: Блок "Система оплаты:" или "Оплата:" в разделе оплаты
  OPTIONS:
    - Поденная
    - Месячная
  RULES:
    - "Поденная ставка", "дниówka", "за день", "в день", "/день", "/dzien" → Поденная
    - "Ежемесячная", "Раз в месяц", "в месяц", "за месяц", "месячная зарплата" → Месячная
    - Если указана дневная ставка без явного указания типа → Поденная
    - Если указана месячная зарплата без явного указания типа → Месячная
    - Если не указано → null
  USAGE: Определение типа оплаты для правильной интерпретации полей зарплаты
  EXAMPLES:
    "Система оплаты: Поденная ставка" → Поденная
    "Система оплаты: Ежемесячная" → Месячная
    "450 злотых нетто за день" → Поденная
    "3000 евро за непрерывную работу" (месячная) → Месячная
    "12 000 злотых за полный месяц работы" → Месячная
  NOTE: Это поле определяет, как интерпретировать значения в полях зарплаты

FIELD_8: Минимальная зарплата (нетто)
  TYPE: number
  DESCRIPTION: Минимальная ставка оплаты (нетто) в валюте, указанной в поле "Валюта зарплаты". Единица измерения зависит от типа оплаты (день или месяц)
  SOURCE: Блок "Оплата:", "Средняя дневная зарплата:" или "Средняя месячная зарплата (нетто):" в разделе оплаты
  FORMAT: Число с возможностью десятичных значений
  RULES:
    - Извлечь число из текста
    - Если диапазон (300-320) → взять минимальное значение
    - Для поденной оплаты: извлекать из "Средняя дневная зарплата:" или "Поденная ставка"
    - Для месячной оплаты: извлекать из "Средняя месячная зарплата (нетто):" или "месячная зарплата"
    - Если тип оплаты не определен → попытаться определить по контексту
  USAGE: Сортировка и фильтрация по минимальному уровню оплаты
  EXAMPLES:
    Поденная: "Поденная ставка – 450 злотых нетто" → 450 (валюта: PLN, тип: Поденная)
    Поденная: "300-320 злотых нетто" → 300 (валюта: PLN, тип: Поденная)
    Поденная: "95€/день" → 95 (валюта: EUR, тип: Поденная)
    Месячная: "3000 евро за непрерывную работу" → 3000 (валюта: EUR, тип: Месячная)
    Месячная: "12 000 злотых за полный месяц работы" → 12000 (валюта: PLN, тип: Месячная)
  NOTE: Единица измерения (день/месяц) определяется полем "Тип оплаты". Валюта указывается в поле "Валюта зарплаты"

FIELD_9: Максимальная зарплата (нетто)
  TYPE: number
  DESCRIPTION: Максимальная ставка оплаты (нетто) в валюте, указанной в поле "Валюта зарплаты". Единица измерения зависит от типа оплаты (день или месяц)
  SOURCE: Блок "Оплата:", "Средняя дневная зарплата:" или "Средняя месячная зарплата (нетто):" в разделе оплаты
  FORMAT: Число с возможностью десятичных значений или null
  RULES:
    - Извлечь максимальное число из диапазона
    - Заполняется только при наличии диапазона
    - Если указана фиксированная сумма → null
  USAGE: Сортировка и фильтрация по максимальному уровню оплаты
  EXAMPLES:
    Поденная: "300-320 злотых нетто" → 320 (валюта: PLN, тип: Поденная)
    Поденная: "400-450 злотых нетто" → 450 (валюта: PLN, тип: Поденная)
    Поденная: "450 злотых нетто" → null (фиксированная сумма)
    Месячная: диапазон не встречается в реальных данных → обычно null
  NOTE: Заполняется только при наличии диапазона зарплаты. Если указана фиксированная сумма - null

FIELD_10: Валюта зарплаты
  TYPE: select
  DESCRIPTION: Валюта, в которой указана зарплата в полях "Минимальная зарплата" и "Максимальная зарплата"
  SOURCE: Блок "Оплата:", "Средняя дневная зарплата:" или "Средняя месячная зарплата (нетто):" в разделе оплаты
  OPTIONS:
    - PLN
    - EUR
  RULES:
    - "€", "EUR", "евро", "euro" → EUR
    - "zł", "PLN", "злотых", "зл" → PLN
    - Если зарплата не указана → null
  USAGE: Фильтрация и понимание валюты зарплаты
  EXAMPLES:
    "450 злотых нетто" → PLN
    "95€/день" → EUR
    "3000 евро" → EUR
    "12 000 злотых" → PLN
    Зарплата не указана → null
  NOTE: Заполняется только если указана зарплата. null = зарплата не указана

FIELD_13: Тип экипажа
  TYPE: select
  DESCRIPTION: Требуемый тип экипажа для работы
  SOURCE: Текст вакансии, раздел условий работы или требований
  OPTIONS:
    - соло
    - парный
  RULES:
    - "Одиночный", "один водитель", "соло", "Solo" → соло
    - "Парный экипаж", "в двойке", "два водителя", "семейный экипаж", "муж и жена" → парный
    - Не указано или "любой экипаж" → null (пустое = любой)
  USAGE: Матчинг с кандидатами по типу экипажа. null = подходит любой экипаж
  EXAMPLES:
    "Требуется парный экипаж" → парный
    "Семейный экипаж приветствуется" → парный
    Не указано → null
  NOTE: Семейный экипаж = парный экипаж (два водителя). null в Notion = нет ограничений

FIELD_14: Требование польского языка
  TYPE: select
  DESCRIPTION: Требование к владению польским языком
  SOURCE: Текст вакансии, раздел требований к кандидату
  OPTIONS:
    - Обязательно
    - Желательно
  RULES:
    - "Польский обязателен", "требуется знание польского" → Обязательно
    - "Польский желателен", "знание польского приветствуется", "разговорный уровень" → Желательно
    - "Не требуется", "не обязательно", не упоминается → null (пустое = не требуется)
  USAGE: Матчинг с кандидатами по владению польским языком. null = польский не требуется
  EXAMPLES:
    "Требуется знание польского языка" → Обязательно
    "Знание польского будет плюсом" → Желательно
    "Знание польского не обязательно" → null
    Не упоминается → null
  NOTE: null в Notion = нет требования к польскому языку

FIELD_15: Город базы
  TYPE: text
  DESCRIPTION: Город или регион, где находится база/стоянка
  SOURCE: Название вакансии, блок "База:", "Место работы (база):", "Lokalizacja:", адрес компании
  FORMAT: Текстовое значение (название города ТОЛЬКО на польском языке, латиница)
  RULES:
    - Извлечь название города из блока "Место работы (база):" или названия вакансии
    - ВАЖНО: Если город указан на русском (кириллица) → ПЕРЕВЕСТИ на польский (латиница)
    - Убрать индекс и лишние слова, оставить только название города
    - Если указан только регион → использовать регион
    - Если не указано → null
  USAGE: Матчинг с кандидатами по предпочтительной локации базы
  EXAMPLES:
    "Место работы (база): 34-240 Йорданув" → Jordanów
    "Место работы (база): Краков" → Kraków
    "Место работы (база): Лодзь" → Łódź
    "База: Варшава" → Warszawa
    Название: "Szczecin (тент)" → Szczecin
    "Lokalizacja: Poznań" → Poznań
    Не указано → null
  NOTE: Важный фактор - кандидаты часто отказываются из-за неудобной локации базы

FIELD_16: Допустимое гражданство
  TYPE: multi_select
  DESCRIPTION: Список гражданств, которые допускаются для данной вакансии (только эти)
  SOURCE: Текст вакансии, раздел требований к кандидату
  FORMAT: Массив строк с полными названиями стран на РУССКОМ языке
  RULES:
    - Если указано "только граждане Украины" → ["Украина"]
    - Если указано "граждане СНГ" → ["Украина", "Беларусь", "Молдова", "Казахстан", "Узбекистан", "Таджикистан", "Грузия", "Азербайджан", "Армения", "Кыргызстан"]
    - Если указано "только граждане Украины и Беларуси" → ["Украина", "Беларусь"]
    - "русскоязычные", "русскоговорящие", "для русскоязычных" → ["Украина", "Беларусь"]
    - Если нет ограничений или не указано → [] (пустой массив = без ограничений)
    - ВАЖНО: Полные названия стран на РУССКОМ языке
  USAGE: Фильтрация вакансий по допустимым гражданствам. [] = подходят все
  EXAMPLES:
    "Только для граждан Украины" → ["Украина"]
    "Принимаем граждан Украины, Беларуси" → ["Украина", "Беларусь"]
    "Для русскоязычных водителей" → ["Украина", "Беларусь"]
    Не указано → []
  NOTE: Пустой массив [] = нет ограничений по гражданству, подходят все кандидаты

FIELD_17: Исключённое гражданство
  TYPE: multi_select
  DESCRIPTION: Список гражданств, которые НЕ допускаются для данной вакансии (кроме этих)
  SOURCE: Текст вакансии, раздел требований к кандидату
  FORMAT: Массив строк с полными названиями стран на РУССКОМ языке
  RULES:
    - Если указано "без граждан РФ" → ["Россия"]
    - Если указано "кроме граждан Румынии" → ["Румыния"]
    - Если указано "не принимаем из России и Беларуси" → ["Россия", "Беларусь"]
    - Если нет исключений или не указано → [] (пустой массив = нет исключений)
    - ВАЖНО: Полные названия стран на РУССКОМ языке
  USAGE: Фильтрация вакансий по исключённым гражданствам. [] = нет исключений
  EXAMPLES:
    "Без граждан РФ" → ["Россия"]
    "Не принимаем граждан России и Румынии" → ["Россия", "Румыния"]
    Не указано → []
  NOTE: Пустой массив [] = нет исключений, подходят все кандидаты

ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ ФИЛЬТРОВ:

EXAMPLE_1: Найти водителей с правами CE

ВАЖНЫЕ ЗАМЕЧАНИЯ:

ЗАРПЛАТА:
  - Поле "Тип оплаты" определяет единицу измерения (Поденная = за день, Месячная = за месяц)
  - Поля "Минимальная зарплата" и "Максимальная зарплата" - универсальные, работают для обоих типов оплаты
  - Валюта указывается в отдельном поле "Валюта зарплаты" (PLN, EUR, null = не указано)
  
  ПОДЕННАЯ ОПЛАТА:
  - Если указана фиксированная сумма (например, "450 злотых нетто за день") - заполняется: тип = Поденная, минимальная = 450, максимальная = null, валюта = PLN
  - Если указан диапазон (например, "300-320 злотых нетто") - заполняется: тип = Поденная, минимальная = 300, максимальная = 320, валюта = PLN
  - Если зарплата в EUR (например, "95€/день") - заполняется: тип = Поденная, минимальная = 95, максимальная = null, валюта = EUR
  
  МЕСЯЧНАЯ ОПЛАТА:
  - Если указана месячная зарплата (например, "3000 евро за непрерывную работу") - заполняется: тип = Месячная, минимальная = 3000, максимальная = null, валюта = EUR
  - Если указана месячная зарплата (например, "12 000 злотых за полный месяц работы") - заполняется: тип = Месячная, минимальная = 12000, максимальная = null, валюта = PLN
  - Диапазоны месячной оплаты встречаются редко, обычно фиксированная сумма
  
  ОПРЕДЕЛЕНИЕ ТИПА ОПЛАТЫ:
  - Если указано "Система оплаты: Поденная ставка" или "Поденная ставка" → тип = Поденная
  - Если указано "Система оплаты: Ежемесячная" или "Раз в месяц" → тип = Месячная
  - Если указана дневная ставка ("за день", "/день", "дниówka") → тип = Поденная
  - Если указана месячная зарплата ("в месяц", "за месяц", "месячная зарплата") → тип = Месячная
  - Если не указано → тип = null, все поля зарплаты = null, валюта = null

РЕГИОНЫ:
  - Список регионов расширен на основе анализа реальных вакансий
  - Можно выбрать несколько регионов для одной вакансии
  - "Европа (общее)" и "По всей Европе" - для вакансий без конкретных стран
  - ВАЖНО: Поле multi_select позволяет добавлять любые страны/регионы через API, даже если их нет в базовом списке. Notion автоматически создаст новую опцию при обновлении.

КАРТА ВОДИТЕЛЯ:
  - Отдельное поле от Код 95, так как это разные документы
  - Карта водителя нужна для работы с тахографом
  - Код 95 - это свидетельство квалификации водителя

ОБРАБОТКА ДАННЫХ:
  - Если блок не найден → оставить поле пустым (null)
  - Если значение "Не указано" или "Информация отсутствует" → null
  - Если несколько вариантов в тексте → выбрать наиболее точный
  - Если значение не совпадает с опциями → попытаться найти ближайшее соответствие
  - Если не найдено → оставить пустым
  - Для multi_select полей → добавить все найденные значения
  - Для select полей → выбрать первое или наиболее подходящее

ГРАЖДАНСТВО:
  - Два отдельных поля: "Допустимое гражданство" и "Исключённое гражданство"
  - Допустимое гражданство: список стран, граждане которых МОГУТ работать (белый список)
  - Исключённое гражданство: список стран, граждане которых НЕ МОГУТ работать (чёрный список)
  - Оба поля - массивы строк с названиями стран на РУССКОМ языке
  - Пустой массив [] = нет ограничений/исключений
  - При матчинге это БЛОКИРУЮЩИЕ факторы:
    * Если допустимое гражданство НЕ пустое и кандидата нет в списке → блок
    * Если исключённое гражданство содержит гражданство кандидата → блок
  - "русскоязычные" = ["Украина", "Беларусь"] (типичная интерпретация)
  - "граждане СНГ" = полный список стран СНГ

ПОЛЯ ДЛЯ МАТЧИНГА (FIELD_13-17):
  
  ТИП ЭКИПАЖА:
  - Если в вакансии указано "парный экипаж" или "семейный экипаж" → "парный"
  - Если указано "соло" или "один водитель" → "соло"
  - Если не указано → null (пустое поле в Notion = подходит любой экипаж)
  - При матчинге: null = нет ограничений, любой кандидат подходит
  - При фильтрации в Notion: "парный ИЛИ пусто" покажет вакансии для парных и без ограничений
  
  ТРЕБОВАНИЕ ПОЛЬСКОГО:
  - Если требуется обязательно → "Обязательно"
  - Если желательно/разговорный уровень → "Желательно"
  - ВАЖНО: Если указан выбор из нескольких языков ("русский, польский ИЛИ английский") → null (достаточно любого)
  - Если не требуется или не упоминается → null (пустое = не требуется)
  - При матчинге: null = польский не нужен, любой кандидат подходит
  
  ГОРОД БАЗЫ:
  - Извлекать из названия вакансии (часто в начале: "Szczecin (тент)")
  - Если указан блок "База:" или "Lokalizacja:" → использовать его
  - ВАЖНО: Название города ТОЛЬКО на польском (латиница), переводить с русского
  - При матчинге: сравнивать с preferred_base_cities кандидата (если указано)
  - Если город далеко от предпочтений кандидата → warning (не блокер)
  
  ДОПУСТИМОЕ ГРАЖДАНСТВО:
  - Если указаны конкретные страны ("только граждане Украины") → массив стран
  - Если нет ограничений или не указано → [] (пустой массив)
  - При матчинге: [] = нет ограничений, любой кандидат подходит
  - При матчинге: если массив НЕ пустой → проверить, что гражданство кандидата в списке
  - Блокирующий фактор: кандидат НЕ подходит, если его гражданства нет в списке допустимых
  
  ИСКЛЮЧЁННОЕ ГРАЖДАНСТВО:
  - Если указаны исключения ("без граждан РФ") → массив исключённых стран
  - Если нет исключений или не указано → [] (пустой массив)
  - При матчинге: [] = нет исключений, любой кандидат подходит
  - При матчинге: если массив НЕ пустой → проверить, что гражданства кандидата НЕТ в списке
  - Блокирующий фактор: кандидат НЕ подходит, если его гражданство в списке исключённых
